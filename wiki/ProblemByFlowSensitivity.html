Take the following example: 

public:=1; temp :=0;
if secret then temp:=1;
if temp != 1 then public:=0

If secret=0, then temp=0 and public=0 in the end. If secret=1, then temp=1 and public=1 in the end. For the dynamic enforcement for the flow-sensitive security, the security checking is enforced only when the assignment occurs. In this example, for the second conditional, if temp=0 before the conditional executes, then public:=0 is not checked since this branch does not execute although temp becomes high and public is still low.
