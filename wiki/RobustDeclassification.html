Introduction

This paper introduces a property to describe what kind of declassification is fine. With the corresponding, a property called Robust Declassification is proposed. Robust Declassification says that an active attacker cannot aquire more information for the program than the passive attacker, even if the program contains declassification statement. Robust Declassification defines a system is robust with repect to what class of active attackers. This paper does not propose a method to enforce Robust Declassification, but proves that a secure program(which satisfies the noninterference) satisfies the robust declassification, which means that program combined by two secure programs is still secure.

Some Understanding

For the example in Sec3, the attacker adds some transitions to the system:

\[ \langle 0,h,p,q,r \rangle \mapsto_A \langle 0,h,h,q,r \rangle \]

which means that copy the secret \(h\) into the password \(p\). The observational equivalence induced on the attacker system \(S'\)

\[ \begin{array}{c} \langle t,h,p,q,r \rangle S'[\approx] \langle t',h',p',q',r' \rangle \\ \Leftrightarrow \\ (t=t') \wedge (q=q') \wedge (r=r') \wedge \\ (t=0 \Rightarrow p=p' \wedge h=h' \wedge p=h' \wedge h=p') \end{array} \]

The reason why the equivalent condition is

\[ (t=t') \wedge (q=q') \wedge (r=r') \wedge (t=0 \Rightarrow p=p' \vee h=h' \vee p=h' \vee h=p') \]

instead of

\[ (t=t') \wedge (q=q') \wedge (r=r') \wedge (t=0 \Rightarrow p=p' \wedge h=h') \]

is that the attacker cannot judge that whether the secret \(h\) is copied into \(p\).

Formally,for the nondeterministic system, two runs from \(\sigma_1\) and \(\sigma_2\) are equivalent iff for the trace \(\tau_1\) starting from \(\sigma_1\), there is a trace \(\tau_2\) starting from \(\sigma_2\) such that \(\tau_1 \approx \tau_2\) and vice versa.

A more savvy attacker might also toggle \(r\) whether he copied \(h\) into \(p\), thus indicating that \(p\) does in fact contain \(h\). This smarter attack adds these transitions:

\[ \begin{array}{l} \langle 0,h,p,q,0 \rangle \mapsto_A \langle 0,h,h,q,1 \rangle \\ \langle 0,h,p,q,1 \rangle \mapsto_A \langle 0,h,h,q,0 \rangle \end{array} \]

The equivalence relation induced by \(S'\) now is given by:

\[ \begin{array}{l} \langle t,h,p,q,r S[\approx] \langle t',h',p',q',r' \\ \Leftrightarrow \\ (t=t') \wedge (q=q') \wedge (r=r') \wedge (t=0 \Rightarrow p=p' \vee h=h') \end{array} \]

The attacker can learn more detailed information about \(h\).

The Motivation

For the program which allows the declassification, an passive attacker is allowed to observe some secret information. But the declassification mechanism may be abused by the Active attacker.

Active Attacks: attackers are able to change the behavior of the executing system. For example, in a system that is a single-computer program, the attacker might overwrite memory locations or registers of the machine.

Passive Attacks: attacker can make(perhaps limited) observations of the system and draw inference from those observations.

Take the following program,

if (user = iuser && passwd = ipasswd) {

output_to_public ("login on OK")

} else {

output_to_public ("login on fail")

}

The program just leak the user and passwd to the public. But if the attacker(Active Attacker) can change the state of the program. for example, add the statement passwd:=h where \(h\) is secret. Then some information about \(h\) is leaked.

The goal for the Robust Declassification is to ensure that an active attacker cannot get more information than the passive attacker.

About Declassification

The starting point for the notion of declassification is that any system that leaks information -- any system that does not satisfy \(SP(\approx)\) -- can be thought of as containing declassifications. A passive attacker may be able to learn some information by observing the system but, by assumption, that information leakage is allowed by the security policy.
