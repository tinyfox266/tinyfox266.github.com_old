Introduction

This paper proposes a type system to enforce Robust Declassification. Additionally, this paper defines the a language-based definition of active attacker and a language-based definition of Robust Declassification.

This paper connects the confidentiality and the integrity of the program.

Basic Idea

The attacker cannot influence the decision of the declassification.

Take the following example: 

[.]; if x_LL then y_LH:=declassify(z_HH,LH)
               else Skip

is not robust, because the attacker might affect what is declassifieid or when it is declassified, by controlling the decision variable x_LL.

So only high-integrity variable(which cannot be modified by the attacker) can emerge in the guard condition of the conditional and the loop. And attacker can only modify the variable which is LL(Low confidentiality, Low integrity).

Active Attack

Active attack is modelled by a language which can only manipulate on the LL(Low confidentiality, Low integrity) variables.

The formal definition is shown below:

\[ \begin{array}{lll} a & ::= & \mbox{skip} \\ & ~|~ & v:=e(\forall v \in \textit{Vars}(e).\Gamma(x)=l\Gamma(v)) ~|~ a_1;\ a_2 \\ & ~|~ & \mbox{if } b \mbox{ then } a_1 \mbox{ else } a_2 (\forall x \in \textit{Vars}(b).\Gamma(x)=l) \\ & ~|~ & \mbox{while } b \mbox{ do } a (\forall x \in \textit{Vars}(b).\Gamma(x)=l) \end{array} \]

Program

A program is composed by two parts: (i). high-integrity context, which is original program that may contain the declassification statement; (ii). low-integrity code, which is the active attack code.

The formal definition of high-integrity context is shown below:

\[ \begin{array}{ll} e ::= \textit{val} ~|~ v ~|~ e_1 \mbox{ op } e_2 ~|~ \mbox{declassify}(e,l) \\[2ex] c[\vec{\bullet}] ::= \mbox{skip} ~|~ v:=e ~|~ c_1;\ c_2 ~|~ \mbox{if } e \mbox{ then } c_1 \mbox{ else } c_2 ~|~ \mbox{while } e \mbox{ do } c ~|~ \bullet \end{array} \]

Note that the robustness is really a property of a high-integrity program context rather than of an entire program.

Security Type System for Robustness

The security type system is used to restrict the high integrity context. The goal for the Robust Declassification is to prevent the declassification mechanism from abusing by the Active Attacker.

The key rule governing uses of declassification states that only high-integrity data is allowed to be declassified and that declassification might only occur in a high-integrity context.

The formal definition of the declassification typing rules is shown below:
